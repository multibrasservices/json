Comprendre le Format JSON (JavaScript Object Notation)

Le JSON (JavaScript Object Notation) est un format d'échange de données léger, largement utilisé pour la communication entre les applications web (clients) et les serveurs (APIs). Il est facile à lire et à écrire pour les humains, et facile à analyser et à générer pour les machines.

1. Qu'est-ce que le JSON et pourquoi est-il essentiel ?

Le JSON est devenu le standard de facto pour les APIs RESTful et les applications modernes en raison de ses nombreux avantages :

  • Légèreté : Moins verbeux que d'autres formats comme le XML.
  • Simplicité : Sa syntaxe est minimale et intuitive.
  • Lisibilité : Sa structure est claire et facile à comprendre.
  • Interopérabilité : Indépendant du langage de programmation, il est supporté par presque tous les langages.
  • Flexibilité : Permet de représenter des données simples ou des structures complexes.

2. Les Deux Structures Fondamentales du JSON

Le JSON est construit à partir de deux structures de base : les objets et les tableaux.

a) Les Objets `{}` (Collections de paires clé-valeur)

Un objet JSON représente une collection non ordonnée de paires clé-valeur.

  • Délimiteurs : Commence et se termine par des accolades `{}`.
  • Paires clé-valeur : Chaque objet contient une ou plusieurs paires clé-valeur, séparées par des virgules `,`.
  • Clé : Doit toujours être une chaîne de caractères (texte) et entourée de guillemets doubles `""`.
  • Valeur : Peut être de n'importe quel type JSON (voir section 3), séparée de la clé par un deux-points `:`.

Exemple d'objet :

{
  "idProduit": "PROD-XYZ-123",
  "nom": "Smartphone Ultra",
  "prix": 799.99,
  "disponible": true
}

b) Les Tableaux `[]` (Listes ordonnées de valeurs)

Un tableau JSON représente une liste ordonnée de valeurs.

  • Délimiteurs : Commence et se termine par des crochets `[]`.
  • Valeurs : Contient une ou plusieurs valeurs, séparées par des virgules `,`.
  • Types de valeurs : Les valeurs peuvent être de n'importe quel type JSON, y compris d'autres objets ou tableaux.

Exemple de tableau de chaînes de caractères :

[
  "pomme",
  "banane",
  "orange",
  "kiwi"
]

Exemple de tableau d'objets (très courant) :

[
  {
    "idCommande": "CMD-001",
    "client": "Alice",
    "total": 120.50
  },
  {
    "idCommande": "CMD-002",
    "client": "Bob",
    "total": 55.00
  }
]

3. Les Types de Valeurs Autorisés en JSON

Une valeur dans une paire clé-valeur ou dans un tableau peut être de l'un des types suivants :

  • String (Chaîne de caractères) : Texte entouré de guillemets doubles. Ex: "Ceci est un texte", "2025-08-29T10:00:00Z"
  • Number (Nombre) : Entier ou décimal. Ex: 123, 3.14, -50, 0.001
  • Boolean (Booléen) : true ou false (sans guillemets).
  • Null : null (représente l'absence intentionnelle de valeur, sans guillemets).
  • Object (Objet) : Une autre structure {}.
  • Array (Tableau) : Une autre structure [].

Exemple combinant plusieurs types :

{
  "produit": {
    "nom": "Ordinateur Portable",
    "marque": "TechCorp",
    "caractéristiques": [
      "Écran 15 pouces",
      "8GB RAM",
      "256GB SSD"
    ]
  },
  "quantité": 2,
  "prixUnitaire": 850.00,
  "enStock": true,
  "dateLivraisonEstimée": null
}

4. Règles de Syntaxe Essentielles et Erreurs Courantes

Le JSON est un format strict. La moindre erreur de syntaxe le rend invalide.

  • Guillemets Doubles Obligatoires pour les Clés et les Chaînes :
    • Correct : {"clé": "valeur"}
    • Incorrect : {clé: "valeur"}, {'clé': 'valeur'}, {"clé": valeur}
  • Pas de Virgule Finale :
    • Correct : {"a": 1, "b": 2}
    • Incorrect : {"a": 1, "b": 2,}
    • Correct : [1, 2]
    • Incorrect : [1, 2,]
  • Pas de Commentaires : Le JSON ne supporte PAS les commentaires (// ou /* */). Si vous en incluez, le JSON sera invalide.
  • Sensibilité à la Casse : Les clés sont sensibles à la casse ("Nom" est différent de "nom").
  • Encodage : Le JSON doit être encodé en UTF-8.

5. Validation JSON

Pour vérifier la validité de votre JSON, utilisez des outils de validation en ligne (ex: jsonlint.com, jsonformatter.org`). Ils vous aideront à identifier rapidement les erreurs de syntaxe.
